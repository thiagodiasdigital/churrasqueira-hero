  ---
  Não precisa de tamanho separado para mobile

  O projeto não usa next/image — usa divs com aspect ratio CSS (aspect-[4/3], aspect-[3/4]). O container é responsivo e redimensiona
  automaticamente. Então:

  - Uma única imagem por slot é suficiente — o CSS cuida do crop/fit
  - Formato recomendado: WebP (consistente com o hero)
  - Resolução: basta seguir as dimensões acima (a maior variante) que o CSS escala para baixo em telas menores
